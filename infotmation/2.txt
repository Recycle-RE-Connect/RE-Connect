Java 결혼정보업체 프로젝트 정보구조도 예시:

- 사용자 (User)
  - ID (PK)
  - 비밀번호
  - 이름
  - 생년월일
  - 성별
  - 연락처
  - 주소
  - 결혼 상태 (초혼/재혼)
  - 프로필 ID (FK)

- 프로필 (Profile)
  - ID (PK)
  - 사용자 ID (FK)
  - 관심사
  - 성격 특성
  - 희망 결혼 파트너 조건
  - 매칭 결과 ID (FK)

- 매칭 결과 (MatchResult)
  - ID (PK)
  - 사용자 ID (FK)
  - 상대방 ID (FK)
  - 매칭 점수
  - 채팅 기록 ID (FK)

- 채팅 기록 (Chat)
  - ID (PK)
  - 채팅방 ID (FK)
  - 보낸 사람 ID (FK)
  - 메시지 내용
  - 보낸 시간

- 채팅방 (ChatRoom)
  - ID (PK)
  - 사용자 1 ID (FK)
  - 사용자 2 ID (FK)

- 매칭 알고리즘 설정 (MatchingAlgorithm)
  - ID (PK)
  - 알고리즘 이름
  - 매칭 기준 (프로필 정보의 가중치)
  - 성격 특성 매칭 로직
  - 희망 파트너 조건 매칭 로직

- 데이터베이스 테이블 관계:
  - 사용자 - 프로필 (일대일)
  - 사용자 - 매칭 결과 (일대다)
  - 사용자 - 채팅 기록 (일대다)
  - 채팅방 - 채팅 기록 (일대다)
  - 사용자 - 채팅방 (다대다)
